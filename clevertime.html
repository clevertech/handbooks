Logging in

You only need your @clevertech.biz Google account.

The first time you login to CleverTime, make sure to set your GitHub Username in your Profile, otherwise the app can't automatically add time from GitHub commits and comments (yeah, that's right. It freakin' supports time tracking from comments as well!).

That's it.

Logging Time Automatically

If you're working on a GitHub repo, you can track time from the comfort of your terminal/console (commits), or your web browser (issue comments).

Worked hours can be tracked in the comments or commits, using a simple syntax:

  • [f:1h] will add 1 hour to the issue's worked hours
  • [f:20m] will add 20 minutes to the issue's worked hours
  • [f:1.5h] will add 1 hour and 30 minutes to the issue's worked hours

The hours will be automatically added to the project with the description the same as the comment/commit. If you leave the comment/commit message blank (apart from the [f:1h] time log syntax) then the description will be the same as the issue title.

If you don't mention an issue in the commit (e.g. "#1") the time is still tracked, just not tied in to the issue. Whenever possible you should always mention an issue.

Logging Time Manually

Sometimes you will have to log time that is not part of a GitHub repo and need to log it manually.

You can do it any time you want, tracking as little or as much time you want.

In order to do that, you just need to go to CleverTime, and:

  1. Choose a Project. The one the time is related to.
  2. Type a Description. Go crazy. Just make sure it's understandable what you did with the time you're logging.
  3. Set how many Hours you're logging. We advise multiples of 0.25 (15 minutes), but it's really your call.
  4. If you're logging some other day's work, change the Date for the time you're logging.
  5. Click the big blue button that says Log Time.
  6. You're done.

Viewing Entered Time

Going to the Time Entries page will allow you to view time by choosing a few filters.

Start/End Dates are mandatory, and so is at least a user (yourself by default) or a project.

What about Assembla?!

We want to stop using Assembla in the future.

For now, time will be copied from Assembla into CleverTime, once a week or so, but not vice versa.

If your project is being managed in Assembla (we feel for you), you should probably track your time there.

You should not track your time in both CleverTime and Assembla for the same project.

It's fine to track time for different projects in both places, though, if you feel that's fine.
