CleverHub Handbook

Our simplified Project Management Tool for use alongside GitHub.

The power of GitHub, organized.

Logging in

You only need a GitHub account that has permissions to view Clevertech's repositories (if you're looking for private, otherwise, you can still access the public ones).

That's it.

Organizing

We use labels to organize information

  • est-* labels are used for estimating issues (1h, 2h, 4h, 8h)
  • stt-* labels are used for defining the issue's status (New, In Progress, Test)
  • prt-* labels are used for prioritizing the issues (High, Normal, Low)

Worked hours can be tracked in the comments or commits, using a simple syntax:

  • [f:1h] will add 1 hour to the issue's worked hours
  • [f:20m] will add 20 minutes to the issue's worked hours
  • [f:1.5h] will add 1 hour and 30 minutes to the issue's worked hours

If you need to relate an issue to another (for example, a task for implementing a user story), simply mention it in the description (like "Related to #1", where "1" is the number of the related issue).

Limitations

The limitations we have for how many hours you can estimate, the statuses, and 3 levels of priorities exist on purpose.

In terms of estimates, it means:

  • An issue should never take more than a day to solve. If it takes more, split it up.
  • Estimates are not final, so you shouldn't estimate "3h". If you think something will take 3h, estimate it as 4.

Regarding statuses, it means:

  • A ticket is either not started (new), started (in progress), waiting to be approved (test), or approved (closed).
  • Anything "in between" can be addressed with custom labels. Use them to address the project needs.

As for priorities, it means:

  • A ticket is either very important (high), important (normal), or "nice to have" (low).
  • Like the statuses, anything "in between" can be addressed with custom labels. Use them to address the project needs.
